## 🎯 Приоритет 2 — Массовые (сводные/ОДКР) заявки (заявка 63317) — ✅ сделано

**Контекст:** клиент присылает один отчёт на месяц по многим ТС для сверки;
большинство заявок уже выполнены. ИИ делает по каждому ТС вердикт (Глушение /
Проверить / Данные верны / Нет данных). Клиент требует ОДИН общий ответ по всем.
Реализовано и проверено на проде (63317), 2026-06-19.

- [x] Несколько дат на один объект (О579СХ) — каждая дата отдельной строкой.

### 2.2. Учёт «нет данных на дату, но появились позже» — ✅ через вердикт/формулировку
- [x] Закрыто редактируемым вердиктом + формулировкой «восстановлено <дата>» в общем

### 2.3. Генерация одного общего ответа с группировкой по вердиктам — ✅ сделано
- [x] `POST /issues/{id}/compose_answer` + `compose_aggregate_answer` группирует ТС

## Принцип
ИИ — гибкий в формулировках, но в рамках: решение опирается на **детерминированные
факты** (флаги телеметрии, ситуация с данными, отредактированные вердикты), а LLM
формулирует. Это повышает точность, экономит токены и делает ответ предсказуемым.
